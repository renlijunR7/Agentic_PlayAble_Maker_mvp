GLOBAL DISTRIBUTE · 业务 + 实现

全球分发：从一个定稿 Playable
到 8 平台 × 多地域同时跑量

分发是「营销」这一段的出口——把工作台产出的 Playable 推到各平台、各地域，由 Forge 按时机自动投放、回收数据、再决定扩量或暂停。本文上半部分讲业务流程（谁、做什么、为什么），下半部分对照 distribute.jsx 讲项目里怎么落地的。

01

业务流程 · 5 步

1
来源 · 工作台

定稿入列

  • Playable 在工作台通过体检
  • 选好要分发的变体 (v1.c…)
  • 进入分发队列
2
Forge · 规划

选平台 + 地域 + 时机

  • 按受众画像匹配平台
  • 按时区选最佳投放时段
  • 横竖屏 / 时长自动适配
3
合规 · 闸门

合规复核

  • 风险词 / 平台规则扫描
  • 未成年 / 金融奖励限制
  • 过审才能上线
4
系统 · 投放

多平台并发上线

  • 各平台 API / 队列推送
  • 运营中 / 待审 / 暂停状态
  • 30 秒刷新实时数据
5
回收 · 闭环

回收 → 扩量 / 暂停

  • 完成局 / CTR / 分享 / 收入
  • 表现好 → 一键扩量
  • 数据回流喂回 Forge
关键角色分工：KOL 只在两处拍板——① 选哪个变体去分发、② 是否同意 Forge 规划的「即将分发」与「扩量」。其余（选平台、定时机、适配、合规、投放、回收）全部由 Forge + 系统 自动完成。
02

每个 Playable 的输入与产出

输入 · 一条分发任务带什么

这些字段决定一次投放怎么跑
  • 变体Pump or Dump · v1.c —— 要分发的具体版本
  • 平台TikTok / IG / YouTube / X / FB / Snap … 目标渠道
  • 地域ID / VN / TH —— 语言与时区，决定适配与时机
  • 时机Forge 规划的投放时段（如「今晚 22:00」）
  • 合规风险词 + 平台规则 + 受众限制的检测结果

产出 · 一条分发任务回收什么

实时回流，喂回 Forge 决策
  • 完成局plays —— 多少玩家玩完一局
  • 收入earn —— 该平台带来的净入账
  • CTR点击 / 进入率，衡量钩子强度
  • 分享分享率，衡量传播力（X 高分享 22.4%）
  • 状态运营中 / 待审 / 暂停 —— 决定下一步

04

平台分发状态机

运营中
已上线并回收数据，正常计入完成局与收入。可扩量。
待审
合规复核中 / 平台审核中，未开始投放。过审后转「运营中」。
暂停
人工或风控暂停，数据冻结。可重新启用或下线。
排队中
已规划未到时机（如「今晚 22:00」），到点自动上线。
MVP 取舍：分发是闭环不能砍的「出口」，但 MVP 阶段只需接通 1-2 个真实平台（如 TikTok / X），其余平台用「待审 / 暂停」占位；趋势图与部分指标可半模拟。等留存跑起来再补齐 8 平台与投放时段优化。
