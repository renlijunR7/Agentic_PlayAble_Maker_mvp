MVP SCOPE · 一把尖刀

MVP 只打磨一件事：
把 Agentic 工作台 做到能闭环

MVP 不是"每个模块都做一点",而是一个核心体验做到惊艳 + 周边模块做到刚好够用。工作台是唯一的尖刀——从一句话到能发布、能结算、能回流。其余 9 个模块全部降级为"支撑工作台跑通"的最小形态,能砍的逻辑先砍,能假数据的先假数据。

THE ONE THING
Agentic 工作台:意图 → 共创 → 发布,端到端真实可用
EVERYTHING ELSE
只做"让工作台能跑通"所必需的最小切面
判断标准
砍掉它,工作台还能完成一次完整闭环吗?能 → 砍/简化
01

尖刀 · Agentic 工作台 必须完整

FULL BUILD · 不可妥协

这一个体验,要做到 惊艳

A · 意图捕获

  • 一句话输入 + 建议起手
  • 资料 / 分身 / 模板挂载
  • Forge 简报反问 3 条

B · 共创(主战场)

  • 三栏:对话 / 预览 / 变体
  • v1 生成 + 3-4 变体
  • 对话迭代实时刷新预览

C · 收束发布

  • 体检:完成率/分享率预估
  • 至少 1 条真实发布通道
  • 导出 / 一键分发

D · 回流(轻量)

  • 发布后指标回到工作台
  • 1 条 A/B 建议通知
  • 存为模板

03

怎么排 · 三步走

PHASE NOW · MVP

跑通一次闭环

目标:一个 KOL 能独立走完全程
  • 工作台 A→D 全链路真实可用
  • 钱包提现 + KYC + 审核跟进
  • 1-2 个真实分发通道
  • 其余模块降级为最小支撑
  • 数据可半模拟,体验不能假
PHASE NEXT · v1.x

把支撑做厚

目标:留存与复用起来
  • 模板自抽取 + 复用率
  • 8 平台分发补齐
  • 星图 / 漏斗可视化上线
  • 分身训练闭环
  • 真实回流数据替换模拟
PHASE LATER · v2

放大愿景

目标:规模化与自动化
  • Agent 世界多节点部署
  • 自进化 / 自动 A/B 编排
  • 模板市场 + 分成
  • 链上奖池 / 多通道结算
  • 团队协作工作台
